# Contributing to Nova DSO Tracker

Thank you for considering a contribution to Nova DSO Tracker!

## Before You Write Any Code

**Open a Discussion first.** This is required for anything beyond a trivial bug fix.
PRs that arrive without prior discussion may be closed without review, regardless of
code quality. This protects both your time and mine.

Use GitHub Discussions to:
- Propose a new feature
- Ask whether a change fits the project's direction
- Align on scope and implementation approach before investing effort

## Hard Constraints

These are non-negotiable. PRs that violate them will be closed:

- **DB schema is frozen.** No new columns, tables, or migrations.
- **Single-purpose PRs only.** One feature or fix per PR. Multi-feature PRs will not be reviewed.
- **No scope creep.** Do not refactor, reformat, or restructure code unrelated to your change.
- **YAML import/export round-trip must be preserved** in any change touching models or config.

## AI-Generated Code

If your contribution was substantially generated by an AI coding tool or agent, please
disclose this in the PR description. AI-generated PRs receive extra scrutiny and must
demonstrate that the author understands and has reviewed every change.

## How to Contribute

1. Open a Discussion and get a green light
2. Fork the repository
3. Create a focused feature branch
4. Make your changes — small and surgical
5. Test locally: `docker compose -f docker-compose-dev.yml up --build`
6. Submit a pull request with a clear description of what changed and why

## Code Style

Match the existing structure. The project uses Flask blueprints — new routes belong
in the appropriate existing blueprint unless discussed otherwise.
