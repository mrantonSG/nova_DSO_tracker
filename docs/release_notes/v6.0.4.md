# Nova DSO Tracker v6.0.4

## Bug Fixes & Improvements

- **First-run admin auto-creation** — When multi-user mode is enabled and `users.db` is empty, a default admin user (`admin`/`admin`) is now created automatically on startup via `init_auth()`. This removes the need for manual database bootstrapping.
- **Custom admin credentials via env vars** — `USER_ADMIN_USERNAME` and `USER_ADMIN_PASSWORD` environment variables are now honoured during first-run admin creation, allowing operators to set a custom default admin password. Falls back to `admin`/`admin` when unset.
- **First-run admin race condition fix** — When running with multiple Gunicorn workers in multi-user mode, both workers could simultaneously attempt to create the default admin user on first startup. This caused a UNIQUE constraint violation and left the losing worker's DB session in a broken state. The admin creation logic in `init_auth()` is now wrapped with `IntegrityError` handling: the losing worker rolls back cleanly and silently skips, while the winner proceeds normally.
- **Updated `.env.example`** — The example env file now documents `USER_ADMIN_USERNAME` and `USER_ADMIN_PASSWORD` for multi-user deployments.
- **Tests for bootstrap logic** — Added `TestDefaultAdminCreation` tests covering the `init_auth()` first-run admin creation path, including default credentials, custom env var credentials, and race condition recovery.
