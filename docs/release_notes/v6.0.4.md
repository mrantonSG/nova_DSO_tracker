# Nova DSO Tracker v6.0.4

## Bug Fixes

- **First-run admin race condition fix** — When running with multiple Gunicorn workers in multi-user mode, both workers could simultaneously attempt to create the default admin user on first startup. This caused a UNIQUE constraint violation and left the losing worker's DB session in a broken state. The admin creation logic in `init_auth()` is now wrapped with `IntegrityError` handling: the losing worker rolls back cleanly and silently skips, while the winner proceeds normally.
