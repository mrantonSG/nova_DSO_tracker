# Nova DSO Tracker v6.0.3

## Bug Fixes & Improvements

- **First-run admin auto-creation** - When multi-user mode is enabled and `users.db` is empty, a default admin user (`admin`/`admin`) is now created automatically on startup via `init_auth()`. This removes the need for manual database bootstrapping.
- **Custom admin credentials via env vars** - `USER_ADMIN_USERNAME` and `USER_ADMIN_PASSWORD` environment variables are now honoured during first-run admin creation, allowing operators to set a custom default admin password. Falls back to `admin`/`admin` when unset.
- **Updated `.env.example`** - The example env file now documents `USER_ADMIN_USERNAME` and `USER_ADMIN_PASSWORD` for multi-user deployments.
- **Tests for bootstrap logic** - Added `TestDefaultAdminCreation` tests covering the `init_auth()` first-run admin creation path.
