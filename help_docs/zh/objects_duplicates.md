
#### 查找并合并重复项随着时间推移，您的星表可能积累了同一目标的重复条目（例如您手动导入了"M 42"，却又从星表中添加了"NGC 1976"）。此工具将扫描您的数据库以查找并解决这些冲突。**工作原理**

Nova应用将扫描整个天体库，找出彼此相距在**2.5角分**范围内的天体对。**解决重复项** 当发现重复项时，系统会并排显示两者。您需决定保留哪个版本作为"主条目"，哪个是需合并并删除的重复项。

* **保留A，合并B：** 对象A保留。对象B被删除，但其数据转移至A。* **保留B，合并A：** 对象B保留。对象A被删除，但其数据转移至B。

**我的数据会发生什么变化？** 合并是为保留历史记录设计的**智能流程**。合并对象时：* **日志：** 所有关联被删除对象的成像会话将重新关联至"保留"对象。* **项目：** 所有活跃或历史项目将转移至"保留"对象。
* **定向数据：**已保存的定向参数将被转移。（注：若*两个*对象均有定向数据，则保留"保留"对象的定向参数）。* **笔记：**私人笔记不会丢失！被删除对象的笔记将追加至"保留"对象笔记的末尾。

**提示：**通常建议保留名称最通用者（例如保留"M 31"，合并"仙女座星系"），以便未来检索。
